## 一、当前项目状态总结

### 1.1 总体评估

项目当前处于**第二阶段"内容与玩法填充"中期水平**，核心框架已全面建立，106次提交历史证明了稳健的开发节奏。Godot 4.3项目配置正确，Forward+渲染器，主场景 `title_screen.tscn`，26个Autoload单例已注册。

**关键数据**:
- 代码规模: 81个 .gd 脚本, 77个 .tscn 场景
- 已完成阶段: Phase 1-12 (框架→战斗→装备→战车→内容→NPC→任务→探索→高级→合成→规范化)
- 当前版本: v0.10
- 已修复Bug: 6个关键Bug全部修复 (2026-07-12)
- 着色器: 12个HD-2D效果着色器
- Autoload单例: 26个

## 二、技术选型说明

### 2.1 引擎与渲染

| 项目 | 选型 | 理由 |
|------|------|------|
| 引擎版本 | Godot 4.3 稳定版 | 长期支持，GDScript 2.0 类型提示 |
| 渲染器 | Forward+ | 桌面平台最优画质，支持3D光照/后处理 |
| 脚本语言 | GDScript | 官方推荐，与引擎API深度集成 |
| 碰撞系统 | 2D碰撞层(探索) + 3D碰撞(城市场景) | 探索用2D简化，城市GridMap用3D |

### 2.2 碰撞层设计

| 层级 | 名称 | 用途 |
|------|------|------|
| 1 | player | 玩家角色/战车碰撞体 |
| 2 | npc | NPC交互区域 |
| 3 | enemy | 敌人/赏金首 |
| 4 | terrain | 地形障碍物 |
| 5 | trigger | 事件触发器/区域入口 |
| 6 | treasure | 宝箱交互区域 |
| 7 | projectile | 战斗弹道 |

### 2.3 输入映射（已实现）

| 动作 | 按键 | 用途 |
|------|------|------|
| move_up/down/left/right | W/A/S/D | 角色移动 |
| ui_up/down/left/right | 方向键 | 菜单导航 |
| ui_accept | Enter/Space | 确认 |
| ui_cancel | ESC | 取消/返回 |
| menu | M | 暂停菜单 |
| interact | E/Z | NPC交互 |
| tank_toggle | T | 上下战车 |
| battle_attack/skill/item/defend/escape | A/S/I/D/R | 战斗快捷键 |

## 三、目录结构规范

当前目录结构已在Phase 12完成规范化，符合Godot最佳实践：

```
dreamers/
├── .github/              # GitHub配置
├── addons/               # 插件 (godot-git-plugin)
├── assets/               # 原始数据文件
│   └── data/dialogues/   # NPC对话JSON
├── docs/                 # 项目文档
├── exports/              # 导出目录
├── fonts/                # 字体 (方正黑)
├── music/                # 音频 (BGM .ogg + SFX .wav)
├── resource/             # 美术资源 (精灵/材质/粒子/网格库/主题/TileSet)
├── scenes/               # 场景
│   ├── HUD/fight/        # 战斗场景
│   ├── characters/       # 角色 (hero/enemies/npc)
│   ├── city/             # 城市 (aoduo_base/aoduo_city/wasteland)
│   ├── ui/               # UI (27个场景)
│   └── world/            # 迷宫 (factory/ant_nest/ruins)
├── scripts/              # 脚本
│   ├── autoload/         # 自动加载 (26个单例)
│   ├── components/       # 可复用组件
│   ├── data/             # 数据定义
│   ├── shader/           # 着色器 (12个)
│   ├── system/           # 系统脚本
│   ├── ui/               # UI脚本
│   └── world/            # 迷宫脚本
└── project.godot
```

**待优化项**:
- `resource/` 目录命名可考虑统一为更标准的命名，但当前已稳定，不宜频繁迁移
- `music/` 可拆分为 `audio/bgm/` 和 `audio/sfx/` 更规范

## 四、阶段工作计划

### 第一阶段：核心闭环打通（Week 1，优先级 P0）

> 目标：让玩家能从标题→探索→战斗→结算→返回的完整流程跑通，迷宫关卡可游玩

#### 任务1.1：实现 random_encounter 随机遇敌系统
- **文件**: `scripts/system/random_encounter.gd`
- **内容**:
  - 基于玩家移动步数/距离触发遇敌
  - 根据当前区域读取敌人遭遇表（enemy_data.gd中按区域分类）
  - 昼夜系统影响遇敌率（夜间加成）
  - 战车模式降低遇敌率
  - 遭遇后通过GameFlow切换到战斗场景
- **验收**: 在荒野/奥多市移动可触发战斗，战斗后正确返回

#### 任务1.2：实现 city_explorer 城市探索管理器
- **文件**: `scripts/system/city_explorer.gd`
- **内容**:
  - 管理城市场景中的可交互对象注册表
  - 处理玩家与NPC/宝箱/商店的交互路由
  - 管理城市入口/出口与区域切换
  - 集成NPC交互信号到对话/商店/任务系统
- **验收**: 奥多市中可与3个NPC交互对话，ESC返回世界地图

#### 任务1.3：实现 battle_effects 战斗特效管理
- **文件**: `scripts/system/battle_effects.gd`
- **内容**:
  - 管理战斗中的视觉特效（攻击闪光、伤害数字、状态图标）
  - 技能特效播放（火焰、冰冻、治疗等）
  - 战车主炮/副炮发射特效
  - 特效队列与并行播放
- **验收**: 战斗中攻击有视觉反馈，技能有对应特效

#### 任务1.4：填充废弃工厂迷宫关卡
- **文件**: `scripts/world/abandoned_factory.gd`, `scenes/world/abandoned_factory.tscn`
- **内容**:

  - 放置事件触发器（入口/出口/BOSS房间门）

  - 添加环境叙事元素（损坏的终端、遗留的日志）
  - 设置区域BOSS：mad_tank（赏金首b02）
  - 迷宫内随机遇敌启用
- **验收**: 可从世界地图进入工厂，遇敌战斗，找到宝箱，击败BOSS

#### 任务1.5：填充蚂蚁巢穴迷宫关卡
- **文件**: `scripts/world/ant_nest.gd`, `scenes/world/ant_nest.tscn`
- **内容**:
  - 配置蚂蚁系敌人遭遇表（巨蚁、酸蚁、蚂蚁女王BOSS）
  - 多层迷宫结构（上层/下层）

  - 蚂蚁女王BOSS战（b03_ant_queen）
- **验收**: 完整可游玩的蚂蚁巢穴关卡

#### 任务1.6：填充古代遗迹迷宫关卡

### 第二阶段：战斗系统深化（Week 2，优先级 P1）

> 目标：战斗玩法从"只按攻击"进化到策略性选择

#### 任务2.1：战斗技能面板集成
- **文件**: `scripts/ui/battle_skill_panel.gd`, `scenes/HUD/fight/fight.gd`
- **内容**:

- **验收**: 战斗中可使用技能，效果正确，SP消耗正确

#### 任务2.2：C装置战术技能战斗集成
- **文件**: `scripts/autoload/c_device_system.gd`, `scenes/HUD/fight/fight.gd`
- **内容**:
  - **迎击**：敌人攻击战车时，C装置有概率拦截并反击
  - **援护**：C装置替队友承受伤害（部分减免）
  - **自动归返**：战车大破时自动撤退回基地
  - **目标锁定**：提高下次攻击命中率与暴击率
  - C装置技能在战斗HUD中显示状态
- **验收**: 战车战斗中C装置技能自动触发，UI有反馈

#### 任务2.3：战车战/白刃战区分
- **文件**: `scenes/HUD/fight/fight.gd`, `scripts/autoload/tank_system.gd`
- **内容**:
  - 驾驶战车时进入战车战模式（使用主炮/副炮/C装置）
  - 步行时进入白刃战模式（使用武器/技能）
  - 战车战伤害计算考虑装甲减免
  - 战车大破判定与强制下车
  - 战斗中可切换战斗模式（特定条件下）
- **验收**: 驾车战斗用车载武器，步行战斗用个人武器

#### 任务2.4：队友AI战斗行为
- **文件**: `scripts/autoload/party_manager.gd`, 新增 `scripts/autoload/party_ai.gd`
- **内容**:
  - 机械师AI：优先修复战车，其次治疗
  - 猎人AI：优先攻击弱点，使用陷阱技能
  - 医生AI：优先治疗低HP队友，其次攻击
  - 可设置AI倾向（积极/平衡/保守）
  - 玩家可手动接管队友操作
- **验收**: 队友在战斗中自动行动，行为符合角色定位
